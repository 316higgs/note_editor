# CLAUDE.md — note記事 校正プロジェクト 指示書

## このファイルについて

Claude Codeが起動時に読み込む司令塔ファイル。
校正・執筆作業を開始する前に、必ずこのファイルと参照先ファイルを読み込むこと。

---

## プロジェクト概要

**発信者:** 理学博士（素粒子物理学専攻）・CAEエンジニア・クリスチャン  
**媒体:** note（主コンテンツ）/ X（流入源）/ LINE公式（オフ会動線）  
**発信テーマ:** 宇宙の神秘・物理法則の美しさ、そして「なぜ学ぶのか」という研究者の哲学  
**最終目的:** 3つの問いへの読者自身の思索を促すこと
  1. この宇宙はどのようにしてできあがったのか（→科学で答える）
  2. この宇宙はなんのために創造されたのか
  3. その宇宙に存在する人間はどのように生きるべきか

**発信者のスタンス:**  
「知識を誇示する教師」ではなく「同じ問いを持って宇宙を見つめる、少し先を歩く先輩」

## 校正依頼時のデフォルト動作

ユーザーから記事テキストが渡されたとき、以下の順序で処理すること。

### Step 1 — 事前読み込み（必須）

作業開始前に以下を確認する。

- `datasets/editorial_rules/editorial_policy.md` の全セクション
- `personas/reader/persona_character.md` のペルソナ属性・離脱ポイント・信仰表現への反応
- `personas/author/author_character.md` の文体語彙リスト・発信の戦略的軸

### Step 2 — 構造診断

記事全体を読み、以下の4点を評価する。

1. **導入** — 「連載テーマ」と「今回の問い」が冒頭で明示されているか
2. **展開** — 【具体（歴史・逸話）→ 抽象（数式）→ 思想（哲学）】の流れになっているか
3. **脆弱性の開示** — 筆者自身の挫折・不安のエピソードが中盤に配置されているか
4. **結論** — 物理の解説で終わらず、読者に響く「宇宙と自分の存在の尊さ」や「人生の目的に対する本質的な問いかけ」に着地しているか

### Step 3 — 文体校正

`datasets/editorial_rules/editorial_policy.md` セクション4〜8、および `personas/author/author_character.md` の文体語彙リストに従い、以下の順でチェックする。

- [ ] 語尾統一（です・ます調、文末の不自然な切れ）
- [ ] 顔文字・過剰記号の除去（^_^、！！！ など）
- [ ] 専門用語へのクッション補足
- [ ] 「上から教える」断定表現の除去
- [ ] 信仰表現の主観化（「〜である」→「〜と感じた」「〜と捉えることもできます」）
- [ ] 知識マウント・自己顕示的表現の除去

### Step 4 — 科学的正確性の確認

`datasets/editorial_rules/editorial_policy.md` セクション5に従い確認する。

- 比喩・直感表現の後に正確な物理的解釈が補われているか
- 専門用語に括弧での説明が付いているか
- 冗長な数式・年表はないか（あれば削除または別記事への誘導を提案）

### Step 5 — ペルソナ適合チェック

`personas/reader/persona_character.md` を参照し、以下を確認する。

- 宗教的・信仰的表現が早期に出ていないか（信頼構築前の出現は離脱リスク）
- 読者が「自分の話だ」と感じられる不安・悩みへの言及、または真理の威力・美しさに関する着地があるか
- 響くキーワード・嫌うキーワードリストに照らして問題ある表現がないか

### Step 6 — 出力
 
以下のフォーマットで返答する。
 
```
## 1. 全体レビュー
- 記事全体の良い点
- 改善すべき点
- 発信者らしさが保たれているか（author_character.mdの文体語彙・スタンスを基準に評価）
 
## 2. 校正方針
- どの観点を優先して直したか（editorial_policy.mdのどのルールに基づくかを明記）
- どの表現をあえて残したか（発信者の「静かな熱量」「固有の文脈」に関わるもの）
- どの表現を削った、または弱めたか（理由を1行で添える）
 
## 3. 修正版全文
- noteへ貼り付けられるMarkdown形式
- 見出し、段落、余白を整える
- 元の思想・熱量を壊さない
 
## 4. 人間確認ポイント
- 投稿前に本人が確認すべき点
- 科学的事実確認が必要な箇所
- 表現の強さを最終判断すべき箇所
- 信仰・宗教表現の登場タイミングが適切かどうかの判断（persona_character.mdの離脱トリガーを参照）
```

## 新規記事執筆依頼時のデフォルト動作

ユーザーから「〇〇について記事を書いて」と依頼されたとき、以下を確認してから執筆を開始する。

1. **物理テーマ** — 今回扱う物理現象・法則は何か
2. **人生の問いとの対応** — その物理現象は「3つの問い」のどれとパラレルか
3. **脆弱性エピソード** — 発信者のどの実体験を中盤に配置するか（不明なら確認する）

確認後、`datasets/editorial_rules/editorial_policy.md` セクション10の「新規記事作成時の基本方針」に従って構成し、執筆する。

---

## 絶対に行ってはならないこと

以下は校正・執筆のいかなるフェーズでも禁止する。

- 発信者の「静かな熱量」や「固有の文脈」を平滑化・削除すること
- 信仰表現を全て除去して「無色透明な科学記事」に仕上げること
- ペルソナへの確認なしに記事の構成を大幅に変更すること
- 顔文字・扇情的タイトル・知識マウント表現を残すこと
- 信仰的表現を断定形（「〜である」）のまま残すこと
